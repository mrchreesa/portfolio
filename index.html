Appointments booking app (Calendly clone) is a Full-Stack application build with Next js 13 (app dir), MongoDB and TailwindCSS. Essentially it contains the same functionality as Calendy but with a different UI.
- The Landing page and About Us are demo components with Full Page JS. The real functionality starts next.
- Contact Us is a contact form with form validation which sends an email with the form input as well as saves it to the admin dashboard.... Spam is prevented by crosschecking with the database if there already exists a form submission with the same email or phone number. The user is allowed to submit only one enquiry per email, until the enquiry is marked as done on the database.
- Appointments is where the user can find all the available slots for booking. After selecting the slot, details of the user need to be filled in and once submitted a confirmation email is send to verify the user's email in order to prevent spam. In the email there is a link with JWT token, when clicked the booking is verified and taken off the available slots.
- Admin. In order to access the admin panel the user needs credentials. Currently you can explore the functionality with: Email: admin@mail.com Pass: 123123123
After login in, the admin will receive a session with JWT and will be redirected to the Admin Panel, where on the sidebar can find the following pages:
- Dashboard. This is where all the submitted contact forms are being saved as well as sent to an email.
- Appointments: Currently is being developed, but thats where all the booked appointments from the client side will live.
- Availability. This is where the admin can add available slots for booking. The slots will be displayed on the client side under /appointments.
- Register admin. Registration of admins is situated in the admin panel and is available only for Super Admins. A list of all the registered users is displayed aswell. The credentials are securely stored on the database and the password encrypted with bcrypt. If any of the /admin/* routes are tried to be accessed without a session, the user will be redirected to /login
